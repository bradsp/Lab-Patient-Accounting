# Phase 01-03: Lookup Components

## Objective
Create reusable lookup/autocomplete components for common entity types (Clients, Insurance, Providers, CDM) building on the existing AutocompleteInput pattern.

## Context
@LabOutreachUI/Shared/AutocompleteInput.razor
@Lab PA WinForms UI/UserControls/InsuranceLookup.Designer.cs
@Lab PA WinForms UI/UserControls/ProviderLookup.Designer.cs
@Lab PA WinForms UI/LookupForms/CdmLookupForm.Designer.cs
@Lab PA WinForms UI/LookupForms/ClientLookupForm.Designer.cs

## Tasks

### Task 1: Create ClientLookup Component
**Type:** create
**Files:** `LabOutreachUI/Components/Lookups/ClientLookup.razor`
**Action:**
- Build on AutocompleteInput pattern
- Inject DictionaryService for client data
- Search by client name and mnemonic
- Display: Name (Mnemonic) in dropdown
- Return selected Client model
- Support initial value binding
- Include "New" button option for creating clients

**Verify:** Client search returns matching results, selection returns Client object

### Task 2: Create InsuranceLookup Component
**Type:** create
**Files:** `LabOutreachUI/Components/Lookups/InsuranceLookup.razor`
**Action:**
- Search insurance companies and plans
- Display: Plan Name - Insurance Company
- Support filtering by plan type (A, B, C, etc.)
- Return selected Ins model
- Show plan code in results

**Verify:** Insurance search filters correctly, returns complete plan info

### Task 3: Create ProviderLookup Component
**Type:** create
**Files:** `LabOutreachUI/Components/Lookups/ProviderLookup.razor`
**Action:**
- Search physicians by name and NPI
- Display: Name, NPI, Specialty
- Support filtering by provider type
- Return selected Phy model
- Option to show only active providers

**Verify:** Provider search works by name and NPI

### Task 4: Create CDMLookup Component
**Type:** create
**Files:** `LabOutreachUI/Components/Lookups/CDMLookup.razor`
**Action:**
- Search Charge Master by code and description
- Display: CDM Code - Description (Price)
- Return selected Cdm model
- Show CPT code in results
- Support filtering by department/type

**Verify:** CDM search finds items by code and description

### Task 5: Create DiagnosisLookup Component
**Type:** create
**Files:** `LabOutreachUI/Components/Lookups/DiagnosisLookup.razor`
**Action:**
- Search ICD-10 codes by code and description
- Display: ICD Code - Description
- Support adding multiple selections
- Return Dx code and description
- Handle large dataset efficiently (lazy load)

**Verify:** Diagnosis search handles ICD-10 codes, supports multiple selection

### Task 6: Create Generic LookupModal Component
**Type:** create
**Files:** `LabOutreachUI/Components/Lookups/LookupModal.razor`
**Action:**
- Create modal-based lookup for full-screen search
- Grid display with sorting and filtering
- Select button to confirm selection
- Cancel to dismiss
- Use Bootstrap modal

**Verify:** Modal lookup opens, displays grid, returns selection

## Verification
- [ ] All lookups search correctly
- [ ] Selection returns correct model
- [ ] Two-way binding works
- [ ] Large datasets perform acceptably
- [ ] Keyboard navigation works

## Success Criteria
- Lookup behavior matches WinForms lookup forms
- Consistent API across all lookup components
- Good performance with typical data volumes
- Accessible and keyboard-navigable
