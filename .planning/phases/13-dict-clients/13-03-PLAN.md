**Key facts already verified (do not re-derive):**
- `DictionaryService.GetClient(string clientMnem, IUnitOfWork uow = null)` returns a fully populated
  `Client` (Discounts/ClientType/Mappings included). `DictionaryService.SaveClient(Client client, IUnitOfWork uow = null)`
  updates when `client.Id > 0`, inserts when `Id == 0`, manages its own transaction, returns the saved client.
- `DictionaryService.GetGLCodes(IUnitOfWork uow = null)` returns `List<GLCode>` for the Cost Center dropdown.
- Static lookups on `Dictionaries` (LabBilling.Core.Models): `StateSource`, `ClientType`, `FeeSchedule`,
  `EmrType`, `Counties` — each a `Dictionary<string,string>` suitable for `<select>` options.
- Bill Method is NOT a dictionary — it is a fixed list: "INVOICE", "PATIENT", "PER ACCOUNT".
- `Client` scalar fields for this plan: `ClientMnem` (cli_mnem, business key), `Name` (cli_nme),
  `StreetAddress1/2`, `City`, `State`, `ZipCode`, `Phone`, `Fax`, `ContactEmail` (email), `Contact`,
  `FacilityNo`, `Type` (int → client type), `County`, `ElectronicBillingType` (EMR type, string),
  `GlCode` (cost center, string), `BillMethod` (string), `FeeSchedule` (string), `IsDeleted` (deleted).
- Legacy REQUIRED fields (ClientMaintenanceEditForm validators): ClientMnem, FeeSchedule, Type
  (client type), GlCode (cost center), BillMethod. `ClientMnem` is READ-ONLY once the client exists.
- `Client.Type` is an int; `Dictionaries.ClientType` keys are the numeric type codes as strings — convert
  carefully between the bound `<select>` string value and the int property.
- No FluentValidation validator exists for `Client`. Use Blazor `EditForm` with manual/`DataAnnotations`
  validation for the required fields (do not add a Core-side validator in this plan).
- `_Imports.razor` already imports `LabBilling.Core.Models` and `LabBilling.Core.Services`.
</context>
