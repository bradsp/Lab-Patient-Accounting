  <name>Task 2: Apply SQL-Server-ism fix-ups</name>
  <files>Database/Postgres/schema-fixups.sql, Database/Postgres/schema-tables.sql</files>
  <action>
    Reconcile what pgloader does NOT translate correctly, in `schema-fixups.sql` (or by correcting
    `schema-tables.sql`): IDENTITY columns → `GENERATED { ALWAYS | BY DEFAULT } AS IDENTITY`
    (pick BY DEFAULT so explicit inserts from migrated data still work — relevant to Phase 25);
    computed columns (`chrg.calc_amt`, `age_on_date_of_service`) → `GENERATED ALWAYS AS (expr)
    STORED` (translate the T-SQL expression; if non-deterministic, fall back to a trigger or a view-
    computed column and note it); type edge cases — `datetime2`/`datetime`→`timestamp`,
    `bit`→`boolean`, `money`/`smallmoney`→`numeric(19,4)`, `uniqueidentifier`→`uuid`,
    `varbinary`→`bytea`, `xml`→`xml`, `nvarchar(max)`→`text`; default constraints and `GETDATE()`→
    `now()`/`CURRENT_TIMESTAMP`. Re-apply to a freshly-reset `labbilling` so the DDL proves it
    applies clean from scratch.
  </action>
  <verify>Dropping and re-creating the `labbilling` schema from the committed DDL applies with zero errors; identity + computed columns exist with the right PostgreSQL constructs (spot-check `chrg`).</verify>
