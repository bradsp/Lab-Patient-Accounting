Key facts:
- `CustomSqlDatabaseProvider.cs` exists ONLY to rewrite `OUTPUT INSERTED` (SQL Server error 334).
  PostgreSQL uses native `RETURNING` via the PG provider — so this class is **deleted, not ported**.
- Both `UnitOfWorkMain.cs` (~line 98-108) and `UnitOfWorkSystem.cs` (~line 36-46) call
  `.UsingProvider<CustomSqlMsDatabaseProvider>(...)` — both must switch to the PG provider.
- `AppEnvironment.cs` builds the connection with `SqlConnectionStringBuilder` (lines ~64-104, ~113-154),
  with `Encrypt=false; TrustServerCertificate=true; MultipleActiveResultSets=true` (lines ~38-39, 83, 132).
- Build with **VS MSBuild** (not `dotnet build` — Utilities has a COM ref). Path via vswhere; `/p:Platform=x64`.
- This plan will NOT fully build green on its own (22-04 sweeps the remaining SqlClient usages) — that is expected; the gate here is the wiring is correct, not zero errors.
